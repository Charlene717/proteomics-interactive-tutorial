資料庫搜尋與鑑定

從 MS2 譜到蛋白質名稱——理解 search engine、PSM 評分、target-decoy 與 FDR 如何共同保證鑑定可信度。

From MS2 spectra to protein IDs — how search engines, PSM scoring, target-decoy, and FDR jointly ensure reliable identification.

概念

一、肽段譜匹配 (PSM) 的核心邏輯

實驗 MS2 譜（一堆碎片離子峰）需要與「理論譜」比對才能識別胜肽。具體流程：

  1. 從蛋白質序列資料庫（如 UniProt）in silico 消化生成所有理論胜肽；
  2. 對於每個母離子 m/z 範圍內的候選胜肽，計算其 b/y 碎片離子的理論 m/z；
  3. 用評分函數（如 Hyperscore、XCorr、E-value）比較理論譜與實驗譜，最高分者即為 PSM (Peptide-Spectrum Match)。

注意：PSM 只是「最佳猜測」——即便分數高，也可能匹配到隨機相似度。這就是為何需要統計層面的 FDR 控制。

An experimental MS2 spectrum (a pattern of fragment ion peaks) must be compared to "theoretical" spectra to identify the peptide. The workflow:

  1. In silico digest the protein database (e.g., UniProt) into all theoretical peptides;
  2. For each candidate within the precursor m/z tolerance, compute theoretical b/y fragment ion m/z values;
  3. A scoring function (Hyperscore, XCorr, E-value) compares theoretical vs experimental spectrum; the best match is the PSM (Peptide-Spectrum Match).

Key point: a PSM is only a "best guess" — high scores can still arise by chance. This is why statistical FDR control is essential.

FDR 控制

三、Target-Decoy 策略與 FDR

核心邏輯：在 search 時，把蛋白質資料庫（target）與一個誘餌資料庫（decoy，把序列反轉或亂序）合併送進去搜尋。Target 的高分匹配可能是真的；Decoy 的高分匹配必定是假的（因為 decoy 序列實際上不存在於樣品中）。

計算：

Core logic: During search, combine the target database with a decoy database (reversed or shuffled sequences) and run them together. High-scoring target matches may be real; high-scoring decoy matches are by definition wrong (the decoys don't exist in the sample).

Calculation:

FDR = (# Decoy PSM above threshold) / (# Target PSM above threshold)

逐步降低分數閾值，當 FDR 達到 1% 時即為「1% FDR cutoff」。整個流程後可再用 Percolator（半監督 SVM）重新排序 PSM、提升靈敏度。

Progressively lower the score threshold until FDR ≤ 1% (the "1% FDR cutoff"). Percolator (semi-supervised SVM) can then re-rank PSMs for higher sensitivity.

⚠️
三層 FDR 控制：(1) PSM-level（單一光譜匹配） (2) Peptide-level（合併重複 PSM） (3) Protein-level（合併共享胜肽）。
論文中報導的「1% FDR」通常指 protein-level，但實際分析時三層都應檢驗。
Three-tier FDR control: (1) PSM-level (per-spectrum) (2) Peptide-level (collapsed PSMs) (3) Protein-level (collapsed shared peptides).
Papers usually report "1% FDR" at protein level, but all three tiers should be checked in practice.

                            Acetyl (N-term, +42.01)

[filter]
psm_fdr                   = 0.01
peptide_fdr               = 0.01
protein_fdr               = 0.01
report.razor_peptide      = true
# Comet.params 關鍵欄位
database_name                  = uniprot_human_target_decoy.fasta
decoy_search                   = 2   # concatenated target-decoy
num_threads                    = 16
peptide_mass_tolerance         = 20.0
peptide_mass_units             = 2   # ppm
fragment_bin_tol               = 0.02
search_enzyme_number           = 1   # trypsin/P
allowed_missed_cleavage        = 2
variable_mod01                 = 15.9949 M 0 3 -1 0 0
add_C_cysteine                 = 57.021464
# Inspect FDR with R
library(MSnbase)
psm <- read.delim("psm.tsv")

# Estimate FDR by accumulated decoy count
psm <- psm[order(-psm$Score),]
psm$decoy <- grepl("^DECOY_", psm$Protein)
psm$cum_decoy  <- cumsum(psm$decoy)
psm$cum_target <- cumsum(!psm$decoy)
psm$fdr <- psm$cum_decoy / pmax(psm$cum_target, 1)

# Cut at 1% FDR
threshold <- min(psm$Score[psm$fdr <= 0.01])
message("1% FDR score cutoff = ", threshold)
import pandas as pd

psm = pd.read_csv("psm.tsv", sep="\t")
psm = psm.sort_values("Score", ascending=False).reset_index(drop=True)
psm["is_decoy"] = psm["Protein"].str.startswith("DECOY_")

psm["cum_decoy"]  = psm["is_decoy"].cumsum()
psm["cum_target"] = (~psm["is_decoy"]).cumsum()
psm["fdr"] = psm["cum_decoy"] / psm["cum_target"].clip(lower=1)

cutoff = psm.loc[psm["fdr"] <= 0.01, "Score"].min()
n_pass = (psm["Score"] >= cutoff).sum()
print(f"1% FDR cutoff = {cutoff:.2f}, PSMs passing = {n_pass}")
