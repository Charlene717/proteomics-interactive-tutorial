定量策略

LFQ、TMT、SILAC、iBAQ——四種主流策略的原理、精準度、與選擇邏輯。

LFQ, TMT, SILAC, iBAQ — principles, precision, and how to choose between them.

概覽

為什麼定量比鑑定更難？

鑑定只需「找到」蛋白質的存在；定量則要求「比較」其在不同條件下的相對或絕對丰度。蛋白質的離子化效率、層析行為、訊號響應線性範圍都不同，使得定量遠比 RNA-seq 困難。

四大主流策略可分為兩類：Label-free（LFQ、iBAQ）不加標籤、簡單便宜但變異較大；Label-based（TMT、SILAC、iTRAQ）透過化學或代謝標記提升精準度與多工性，但成本高。

Identification only asks "is it there?"; quantification asks "how much, and how does it compare across conditions?" Proteins differ in ionization efficiency, chromatographic behavior, and dynamic range — making quant much harder than for RNA-seq.

Four mainstream strategies divide into two families: Label-free (LFQ, iBAQ) — no labels, cheap and simple but more variable; Label-based (TMT, SILAC, iTRAQ) — chemical/metabolic labeling for precision and multiplexing, but expensive.

四大策略

一、四種定量策略

🆓

LFQ — Label-Free

每個樣品分別上機；用 MS1 peak area（強度）或 spectral counts 比較相對丰度。MaxLFQ、IonQuant、DIA-NN 都實作精緻的 MS1 比對演算法。
• 優點：成本最低；樣品數量無上限；DIA-LFQ 重現性極佳
• 缺點：批次間變異較大；需依賴 MBR (Match-Between-Runs)

Each sample is acquired separately; compare relative abundance via MS1 peak area or spectral counts. MaxLFQ, IonQuant, DIA-NN implement sophisticated alignment.
• Pros: Cheapest; unlimited samples; DIA-LFQ has excellent reproducibility
• Cons: Larger batch effects; relies on MBR (Match-Between-Runs)

🏷️

TMT / iTRAQ — Isobaric

胺基反應性 isobaric tag 在胜肽 N 端與 K 標記，化學結構相同但 MS2 報告離子質量不同（126–135 m/z 等），同一次 run 可比較 6–18 樣品。
• 優點：多工最高（TMTpro 18-plex）；同一 run 降低變異
• 缺點：Ratio compression（共流出干擾）；MS3 (SPS) 可校正但耗時；需大量起始量

Amine-reactive isobaric tags label peptide N-term and K. Same chemical mass, different MS2 reporter ion m/z (126–135 etc.) — 6–18 samples per run.
• Pros: Highest multiplexing (TMTpro 18-plex); intra-run reduces variability
• Cons: Ratio compression (co-eluting interference); MS3 (SPS) corrects but slow; needs high input

🧬

SILAC — Metabolic

細胞培養時餵食 同位素標記 Lys / Arg（¹³C₆-K、¹³C₆¹⁵N₄-R），質量差異在 MS1 譜出現。Heavy/Light 比值即為相對表達。
• 優點：精準度最高（特別是 PTM 分析）；可在最早階段混合，去除技術變異
• 缺點：僅適用培養細胞；無法應用於人類樣品；最多 3-plex (light/medium/heavy)

Cells are fed isotope-labeled Lys/Arg (¹³C₆-K, ¹³C₆¹⁵N₄-R) during culture, creating mass shifts visible in MS1. Heavy/Light ratios give relative abundance.
• Pros: Highest precision (especially PTMs); samples mixed earliest, removing technical variation
• Cons: Cultured cells only; not for human tissue; max 3-plex (light/medium/heavy)

📐

iBAQ — Absolute

Intensity-Based Absolute Quantification：將蛋白質的所有觀測胜肽 MS1 強度加總，再除以該蛋白「理論上可被偵測的胜肽數」，可估算同一樣品內不同蛋白的相對絕對量。
• 適用：樣品內蛋白丰度比較、化學計量分析
• 限制：仍非真正 absolute；不同物種 / 樣品間不可直接比

Intensity-Based Absolute Quantification: sum MS1 intensities of all observed peptides, divide by the number of theoretically observable peptides, to estimate within-sample absolute abundance.
• Use cases: Within-sample stoichiometry, complex composition
• Limits: Not truly absolute; not directly comparable across samples/species

二、選擇策略的決策樹

🌳 定量策略決策

Q1:
樣品數 ≤ 16 且為深度 PTM 研究？ → 是 → TMTpro + MS3 (SPS)。
Q2:
細胞培養 + 訊號傳遞 / PTM 動態？ → 是 → SILAC。
Q3:
大型 cohort > 30 樣品（如臨床、人類組織）？ → 是 → DIA-LFQ。
Q4:
需要絕對丰度（化學計量學）？ → 是 → iBAQ（同樣品內）或 SureQuant + AQUA（跨樣品）。

Absolute abundance (stoichiometry)? → Yes → iBAQ (within sample) or SureQuant + AQUA (across samples).

MSstats / MaxLFQ 範例

library(MSstats)
library(MSstatsConvert)

# Read FragPipe MSstats input
raw <- read.csv("msstats.csv")

# Convert + normalize
processed <- dataProcess(raw,
  normalization = "equalizeMedians",
  summaryMethod = "TMP",
  censoredInt = "NA",
  MBimpute = TRUE)

# Define contrast: Treated vs Control
contrast <- matrix(c(-1, 1), nrow = 1,
  dimnames = list("Tr-Ctl", c("Ctl", "Tr")))

result <- groupComparison(contrast.matrix = contrast,
                          data = processed)
head(result$ComparisonResult)  # log2FC, pvalue, adj.pvalue
import pandas as pd
import numpy as np

# DIA-NN report: protein matrix at pg-level MaxLFQ
df = pd.read_csv("report.pg_matrix.tsv", sep="\t")
df.set_index("Protein.Group", inplace=True)

# Sample columns are MS run names — log2-transform
mat = np.log2(df.iloc[:, 4:].replace(0, np.nan))
print(f"{mat.shape[0]} proteins × {mat.shape[1]} samples")

# Median normalization
mat_norm = mat.subtract(mat.median(axis=0), axis=1)

# Simple group fold-change
ctrl = mat_norm.filter(like="Ctl_").mean(axis=1)
trt  = mat_norm.filter(like="Tr_").mean(axis=1)
fc = trt - ctrl
top_up = fc.nlargest(20)
print(top_up)
