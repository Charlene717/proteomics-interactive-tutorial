STEP 6 / 9

標準化 (Normalization)

校正技術變異，讓樣品之間真正可比——VSN、Median、Quantile、Cyclic LOESS 的選擇與適用情境。

Correct for technical variation so samples become truly comparable — VSN, median, quantile, cyclic LOESS, and when to use each.

概念

為什麼必須做 Normalization？

同一份樣品分兩個小份上機，得到的訊號總強度可能差異 30–50%。原因包括：注射量微差、LC 柱狀態、ESI 噴霧穩定性、儀器靈敏度漂移。不做 normalization 就比較強度，無異於用兩把不同刻度的尺量身高。

Normalization 的核心假設：「大多數蛋白質的真實表達在條件之間是不變的」（中位數穩定性假設）。標準化的目標就是讓樣品之間的整體分布變得相似。

Two technical replicates of the same sample can differ in total signal by 30–50%, due to injection volume, column condition, ESI stability, and instrument drift. Comparing intensities without normalization is like measuring height with two different rulers.

Core assumption: most proteins do not change between conditions (the median-stability assumption). Normalization aligns the overall distributions across samples.

五大方法

一、五種主流 normalization 方法

📊 Median normalization

每個樣品減去其 log2 強度的中位數。最簡單、最廣泛，假設「中位數是 invariant」。
• 適用：樣品變異小、樣品內變異主流；多數標準工具的預設。
• 限制：不修正 variance heterogeneity，極端 dynamic range 表現一般。

Subtract each sample's median log2 intensity. Simplest, most widely used; assumes the median is invariant.
• Use: low sample-to-sample variation; default in most tools.
• Limit: doesn't correct variance heterogeneity; mediocre at extreme dynamic ranges.

📈 VSN

同時做 log-like 轉換與標準化，使所有強度級別的變異近似一致。低豐度蛋白的 variance 也得到合理估計。
• 適用：2018 benchmark 中表現最佳；DEP 套件預設。
• 限制：計算稍慢；解釋性不如 median 直觀。

Combines log-like transform with normalization so that variance is similar across all intensity levels. Low-abundance proteins get reasonable variance estimates.
• Use: Top-ranked in 2018 benchmark; DEP package default.
• Limit: Slower; less intuitive than median.

📐 Quantile normalization

強制所有樣品的 intensity distribution 變得完全相同。每個 percentile 都對齊到 reference。
• 適用：樣品間 distribution 形狀差異大；分批整合。
• 限制：過於強烈——若樣品間真有生物差異（如疾病組蛋白整體偏低），會被「強行抹平」。

Forces all samples to have identical distributions — each percentile aligned to a reference.
• Use: Strongly differing distributions; batch integration.
• Limit: Too aggressive — real biological differences (e.g., disease-shifted distributions) get erased.

🌊 Cyclic LOESS

逐對樣品做 LOESS 回歸再迭代，修正 intensity-dependent bias。微陣列年代主流。
• 適用：需要修正 dependency 但不希望像 quantile 那樣強硬。
• 限制：對 outlier 敏感；在大樣本量下分布性能下降。

Pairwise LOESS regression iterated cyclically, correcting intensity-dependent bias. A microarray classic.
• Use: Correct dependency without forcing identical distributions.
• Limit: Sensitive to outliers; performance degrades at large sample counts.

🔬 Normics (2022)

新穎方法：利用「variance 與資料內 correlation 結構」識別最不變化的子集作 reference，再進行 robust normalization。
• 適用：樣品間有大規模真實生物差異（如疾病 vs 健康）。
• 限制：較新，工具支援不如 VSN 廣泛。

2022 method: uses variance + intrinsic correlation structure to identify the most invariant subset as reference, then performs robust normalization.
• Use: Large genuine biological differences between groups (disease vs healthy).
• Limit: Newer; less tool support than VSN.

🏷️ TMT 專用: IRS

Internal Reference Scaling：每個 TMT batch 都包含一個 pooled reference channel；用 reference channel 的訊號將其他 channels 拉到同一基準。
• 專為 TMT 多 batch 設計，是當前 TMT 跨批次整合的黃金標準。

Internal Reference Scaling: each TMT batch includes a pooled reference channel; other channels are scaled to that common baseline.
• Designed for multi-batch TMT — the gold standard for cross-batch TMT integration.

決策

二、Normalization 方法選擇

🌳 Normalization 決策樹

Q1:
TMT 多批次資料？ → 是 → IRS + median。
Q2:
LFQ 或 DIA 單批次？ → 是 → VSN（最佳）或 median（簡單可靠）。
Q3:
樣品間 distribution 形狀差異大？ → 是 → 試 Cyclic LOESS；謹慎使用 quantile（會抹掉真實差異）。
Q4:
疾病組與健康組蛋白整體丰度有真實差異？ → 是 → Normics 或 robust median 變體。
Q1:
Multi-batch TMT? → Yes → IRS + median.
Q2:
Single-batch LFQ or DIA? → Yes → VSN (best) or median (simple and reliable).
Q3:
Strongly different distributions across samples? → Yes → Try cyclic LOESS; use quantile cautiously (it erases real differences).
Q4:
Genuine global abundance shift (disease vs healthy)? → Yes → Normics or a robust median variant.
