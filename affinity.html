親合性蛋白質體學 (Affinity Proteomics)

Olink 與 SomaScan——兩大非質譜蛋白質體學平台的原理、特點與如何選擇。

Olink and SomaScan — the two leading non-MS proteomics platforms, their principles, traits, and how to choose.

概念

為什麼需要非質譜的蛋白質體學？

MS-based proteomics 在血漿等高動態範圍樣品有先天限制：白蛋白（Albumin）佔總蛋白 50% 以上，壓抑訊號讓低豐度的 cytokines / hormones（pg-ng/mL）難以偵測。

Affinity proteomics（基於抗體或 aptamer 親合試劑）能高通量同時測量 3000–10000+ 蛋白，含許多 MS 偵測不到的低豐度 biomarker。已成為大規模 cohort（UK Biobank 50,000+ 樣品）的主流。

MS-based proteomics struggles with high-dynamic-range samples like plasma: albumin alone is 50%+ of total protein, suppressing signals for low-abundance cytokines/hormones (pg-ng/mL).

Affinity proteomics (antibody- or aptamer-based) measures 3000–10000+ proteins in parallel — including many low-abundance biomarkers MS misses. It dominates large cohorts (UK Biobank 50,000+ samples).

兩大平台

一、Olink vs SomaScan 的本質區別

🔵

Olink — Proximity Extension Assay (PEA)

原理：使用一對抗體標記不同 DNA oligo 標籤；當兩個抗體都成功結合到同一蛋白時，DNA oligo 形成 hybrid → PCR/NGS 定量。
優勢：「雙重認證」大幅降低非專一性；對 PTM、isoform 較不敏感；specificity 較高。
規模：Olink Explore HT (5400+ 蛋白) / Explore 3072。
佔比：UK Biobank 已採用做 50,000+ 樣品。

Principle: antibody pairs each carrying a unique DNA oligo; when both bind the same protein, the oligos hybridize → PCR/NGS readout.
Strength: "Dual authentication" greatly reduces non-specific signal; less PTM/isoform interference; higher specificity.
Scale: Olink Explore HT (5400+ proteins) / Explore 3072.
Adoption: Used by UK Biobank for 50,000+ samples.

🔴

SomaScan — SOMAmer (aptamer)

原理：使用慢解離率修飾單股 DNA (SOMAmer) 與蛋白結合 → 變性洗去未結合者 → 微陣列 / NGS 定量。
優勢：覆蓋率最廣（v4.1 達 11000+ 蛋白）；精密度佳，動態範圍寬。
限制：單一試劑，無「雙重認證」；對結構性 epitope 較敏感。
佔比：Atherosclerosis Risk in Communities (ARIC) 主要平台。

Principle: Slow-off-rate modified single-stranded DNA (SOMAmer) binds proteins → denaturing wash → microarray/NGS readout.
Strength: Broadest coverage (v4.1 has 11000+ proteins); excellent precision; wide dynamic range.
Limits: Single-reagent — no "dual authentication"; sensitive to structural epitopes.
Adoption: ARIC's primary platform.

Key finding (Pietzner 2023 Nature): Median cross-platform correlation is only r ≈ 0.29. Reasons: (1) different epitopes; (2) SOMAmer cross-reactivity; (3) Olink's dual-antibody vs SomaScan's single reagent. Advice: don't directly compare absolute readouts; integrate via phenotype/GWAS as a common anchor.

決策

二、選擇平台的決策

🌳 平台選擇決策

Q1:
需要深度（> 10k 蛋白）且預算允許？ → 是 → SomaScan v4.1。
Q2:
對 specificity 要求極高（如 clinical biomarker）？ → 是 → Olink。
Q3:
需要絕對定量？ → 是 → 兩者皆為相對值——建議補做 MS PRM/SureQuant validation。
Q4:
要整合多個 cohort（如跨研究）？ → 是 → 採用同一平台避免 r=0.29 的不一致；如已既有混合，使用 genetics-based harmonization。
Q1:
Need depth (> 10k proteins) and budget allows? → Yes → SomaScan v4.1.
Q2:
Specificity is critical (clinical biomarker)? → Yes → Olink.
Q3:
Absolute quantification needed? → Yes → Both are relative — add MS PRM/SureQuant validation.
Q4:
Integrating multiple cohorts? → Yes → Use a single platform to avoid the r=0.29 mismatch; otherwise apply genetics-based harmonization.
分析

三、資料分析特殊性

📊 NPX 與 RFU 都是相對值

Olink 用 NPX (Normalized Protein eXpression, log2 scale)，SomaScan 用 RFU。兩者皆相對於 reference——不直接代表 fmol 或 pg/mL。

Olink uses NPX (Normalized Protein eXpression, log2 scale); SomaScan uses RFU. Both are relative to reference — not directly fmol or pg/mL.

🧬 pQTL 整合

用蛋白質 quantitative trait loci (pQTLs) 作為「分子真值」——若一個變異在 Olink 和 SomaScan 對同一蛋白都產生強 cis-pQTL，則該蛋白測量可信。這是當前跨平台 harmonization 的主流方法。

Use protein quantitative trait loci (pQTLs) as "molecular ground truth" — if a variant produces a strong cis-pQTL on both Olink and SomaScan, the protein measurement is trustworthy. The current go-to cross-platform harmonization approach.

📈 缺失值與 LOD

Olink 與 SomaScan 都有 LOD floor。低於 LOD 的測量被回報為 LOD 值（左截尾），分析時應視為 censored data 或用 Tobit regression。

Olink and SomaScan both have LOD floors. Sub-LOD measurements are reported as the LOD value (left-censored); analyze as censored data or use Tobit regression.

🤝 與 MS 互補

Affinity 強在低豐度血漿 biomarker；MS 強在 PTM、proteoform 與深度蛋白組學。多層次整合 (multi-platform) 是 2024-25 大型 atlas 計畫的標準。

Affinity excels at low-abundance plasma biomarkers; MS excels at PTMs, proteoforms, and deep proteomes. Multi-platform integration is standard for 2024-25 large-scale atlases.

import pandas as pd
import numpy as np
from scipy import stats

# Olink NPX data (long format)
npx = pd.read_csv("olink_explore.csv")

# Pivot to wide
wide = npx.pivot_table(index="SampleID", columns="Assay", values="NPX")

# Drop high-missing assays (LOD floor too often)
wide = wide.loc[:, wide.notna().mean() > 0.7]

# t-test per protein
meta = pd.read_csv("meta.csv", index_col=0)
ctl = wide.loc[meta["group"] == "Ctl"]
trt = wide.loc[meta["group"] == "Tr"]

t, p = stats.ttest_ind(trt, ctl, axis=0, nan_policy="omit")
fc = trt.mean() - ctl.mean()
res = pd.DataFrame({"NPX_diff": fc.values, "pval": p},
                   index=wide.columns)
print(res.sort_values("pval").head(20))
