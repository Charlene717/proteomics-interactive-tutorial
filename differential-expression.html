差異表達分析 (Differential Expression)

limma、MSstats、DEP——三大主流框架的設計哲學、適用情境與最佳實踐。

limma, MSstats, DEP — three frameworks, their design philosophies, use cases, and best practices.

概念

為什麼蛋白質體學不適用簡單的 t-test？

樣品數通常 3–6/組，變異估計極不穩定——某些蛋白看似 p-value 很小，但實際上只是因為偶然觀測到極小 variance。t-test 不會「借用」其他蛋白的資訊。

解決方案是 moderated t-statistics（empirical Bayes 變異收縮）：將每個蛋白的 variance 估計往群體中位數方向「拉一拉」，得到更穩健的 t 統計量。這就是 limma 的核心。

With only 3–6 samples per group, variance estimates are extremely unstable — some proteins look significant only because of an accidentally tiny variance. A plain t-test cannot "borrow" information from other proteins.

The solution is moderated t-statistics (empirical Bayes variance shrinkage): shrink each protein's variance toward the population median to get more stable t-values. This is the core idea of limma.

二、執行 DE 的程式碼

library(limma)
mat <- as.matrix(read.csv("protein_log2.csv", row.names = 1))
group <- factor(c("Ctl","Ctl","Ctl","Tr","Tr","Tr"))

design   <- model.matrix(~0+group)
colnames(design) <- c("Ctl","Tr")
contrast <- makeContrasts(Tr-Ctl, levels=design)

fit  <- lmFit(mat, design)
fit2 <- contrasts.fit(fit, contrast)
fit2 <- eBayes(fit2)            # 關鍵：moderated t

res <- topTable(fit2, number = Inf, sort.by = "P")
head(res)  # logFC, t, P.Value, adj.P.Val, B

# Significant cutoff
sig <- res[abs(res$logFC) > 1 & res$adj.P.Val < 0.05, ]
nrow(sig)
library(MSstats)

import pandas as pd
import numpy as np
from scipy import stats
from statsmodels.stats.multitest import multipletests

df = pd.read_csv("protein_log2.csv", index_col=0)
ctl = df[["C1","C2","C3"]].values
trt = df[["T1","T2","T3"]].values

# Note: SciPy ttest_ind isn't moderated; for moderated t use R/limma via rpy2
t, p = stats.ttest_ind(trt, ctl, axis=1, nan_policy="omit")
log2fc = trt.mean(axis=1) - ctl.mean(axis=1)

# BH correction
_, p_adj, _, _ = multipletests(p, method="fdr_bh")

res = pd.DataFrame({"log2FC":log2fc, "pvalue":p, "padj":p_adj},
                   index=df.index).sort_values("padj")
sig = res[(res["padj"] < 0.05) & (res["log2FC"].abs() > 1)]
print(f"# significant: {len(sig)}")

三、避免常見錯誤

🚨
❌ 不要犯這些錯誤：
① 用未校正的 p-value篩選——3000 個蛋白做 t-test 後沒有 BH 校正，等於拿 5% 假陽性率 × 3000 = 150 個假陽性。
② 樣品數 = 3 卻看到「絕對 t-test」結果——應用 limma 等 moderated 方法。
③ 混合 imputation 與 t-test 但未將「imputation indicator」納入 design matrix。
④ 在 TMT 多 batch 資料未做 IRS 就比較不同 batch。
❌ Don't make these mistakes:
① Filtering by uncorrected p-values — 3000 t-tests at α=0.05 gives 150 false positives without BH correction.
② n = 3 per group with naive t-test — use moderated methods like limma.
③ Imputing + t-testing without flagging "imputation indicator" in the design matrix.
④ Comparing across TMT batches without IRS first.
✅
✅ 推薦流程：
① QC + Normalization + Diagnose missingness (前面 3 步)
② 在 design matrix 加入 batch / sex / age 等 confounders 作 covariate
③ 使用 limma（peptide-level 最強）或 MSstats
④ BH adjustment → adj.p < 0.05
⑤ 若 log2FC 過小（< 0.5）即使顯著也應慎用——考慮 biological relevance
⑥ 將結果送入 enrichment（下一步）
✅ Recommended workflow:
① QC + normalization + diagnose missingness (the prior 3 steps)
② Add confounders (batch, sex, age) to the design matrix as covariates
③ Use limma (peptide-level is strongest) or MSstats
④ BH adjustment → adj.p < 0.05
⑤ Reject tiny effect sizes (|log2FC| < 0.5) even if significant — biological relevance matters
⑥ Pipe results into enrichment (next step)

📝 自我檢測

1. 你只有 3 個 Control + 3 個 Treatment 樣品。為何不該用一般 t-test？

1. You have 3 Control + 3 Treatment samples. Why avoid plain t-test?

A. t-test 不能處理連續變數A. t-test can't handle continuous variables
B. 小樣本下 variance 估計極不穩定，會產生大量假陽性；moderated t（limma）可借用其他蛋白的資訊穩定估計B. Small-sample variance estimates are unstable; moderated t (limma) borrows from other proteins
C. t-test 太快，看起來不專業C. t-test is too fast and looks unprofessional
D. t-test 只能比較 RNAD. t-test only works for RNA

2. 一個蛋白 log2FC = 0.15、adj.p = 0.001。最合理的解讀？

2. A protein has log2FC = 0.15 and adj.p = 0.001. Best interpretation?

A. 強訊號，列為主要 hitA. Strong signal — promote to top hit
B. 統計不顯著，移除B. Not significant, remove
C. 統計顯著但 effect size 太小（< 11% 變化），需要結合生物學上下文判斷C. Significant but effect size is tiny (<11% change); needs biological context to interpret
D. 假陽性D. False positive

3. 你有 6 個樣品中有 2 個來自不同 batch。DE 分析時最佳做法？

3. 6 samples with 2 from a different batch. Best DE practice?

A. 在 design matrix 加入 batch 作為 covariate（如 ~ group + batch）A. Add batch as a covariate in the design matrix (~ group + batch)
B. 刪除 batch B 的 2 個樣品B. Drop the 2 batch-B samples
C. 忽略 batch 差異C. Ignore batch differences
D. 把 batch B 與 A 的樣品平均D. Average batch A and B samples together
