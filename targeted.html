靶向蛋白質體學 (Targeted Proteomics)

從 discovery 到 validation——SRM、MRM、PRM、SureQuant 在生物標誌物驗證中的角色。

From discovery to validation — the role of SRM, MRM, PRM, and SureQuant in biomarker validation.

概念

為什麼需要 targeted？

Discovery MS (DDA/DIA) 一次嘗試「看到所有蛋白」，但對單一蛋白的靈敏度、線性範圍、跨樣品重現性仍有限。當你要做以下事情時，targeted MS 是首選：

  • 驗證 discovery 階段的 biomarker（FDA 等級的 validation）
  • 絕對定量（搭配同位素內標 AQUA / SIS）
  • 追蹤 PTM 隨時間動態
  • 跨 100+ 樣品的臨床定量

核心概念：儀器不再 scan「所有可能 m/z」，而是只關注少數預先選定的胜肽（target peptide）的特定碎片（transition）。這帶來 1000× 以上的靈敏度提升。

Discovery MS (DDA/DIA) tries to "see everything", but sensitivity, linear range, and cross-sample reproducibility for single proteins remain limited. Targeted MS is preferred when you need:

  • Validation of biomarkers (FDA-grade)
  • Absolute quantitation (with AQUA / SIS isotope standards)
  • Tracking PTM dynamics over time
  • Clinical quantitation across 100+ samples

Core idea: The instrument no longer scans every possible m/z — it monitors only pre-selected peptides' specific fragments (transitions). This yields >1000× sensitivity gain.

四種方法

一、SRM / MRM / PRM / SureQuant

🔫 SRM / MRM (Triple Quad)

在 triple quadrupole 上：Q1 篩選母離子 → Q2 碎裂 → Q3 篩選特定 fragment ion。
• 原理：一次只監控少數「Q1 m/z → Q3 m/z」transition pair（通常 3–5 個 per peptide）
• 優點：30+ 年成熟，動態範圍 5+ orders；多 lab 通用
• 限制：單位解析度（unit resolution），對干擾物較敏感

On triple quadrupoles: Q1 selects precursor → Q2 fragments → Q3 selects specific fragment.
• Principle: Monitor only a few "Q1 m/z → Q3 m/z" transition pairs per peptide (typically 3–5)
• Pros: 30+ years mature, 5+ orders dynamic range; cross-lab standard
• Cons: Unit resolution; more sensitive to interference

🎯 PRM

在 Q-Orbitrap 上：Q 選母離子 → HCD 碎裂 → Orbitrap 高解析度同時掃描所有 fragment ions。
• 優點：高解析度 (15–60k) 大幅減少干擾；後處理可彈性選擇 transition
• 限制：速度較 SRM 慢；通量 ~50–100 peptide

On Q-Orbitrap: Q selects precursor → HCD fragments → Orbitrap scans all fragments at high resolution.
• Pros: 15–60k resolution massively reduces interference; transitions chosen post-hoc
• Cons: Slower than SRM; ~50–100 peptides per method

🚀 SureQuant (IS-PRM)

Internal Standard-Triggered PRM：先加入 isotope-labeled internal standard (SIS)，儀器偵測到 SIS 時自動觸發其對應 endogenous peptide 的 PRM scan。
• 優點：多工 > 1000 peptide；無需嚴格 RT scheduling
• 適用：大型 cohort 驗證 (Thermo Orbitrap 平台)

Internal Standard-triggered PRM: add isotope-labeled internal standards (SIS); when SIS is detected, the instrument triggers PRM for the matching endogenous peptide.
• Pros: >1000 peptides multiplexed; no strict RT scheduling needed
• Use: Large cohort validation (Thermo Orbitrap)

📏 AQUA / SIS

合成同位素標記胜肽（¹³C₆-K 或 ¹³C₆¹⁵N₄-R）作為內標，以已知濃度加入樣品。
• 輸出：真正 absolute amount (fmol/µg)
• 限制：每個 target 都需訂製 SIS，成本高

Synthesize isotope-labeled peptides (¹³C₆-K or ¹³C₆¹⁵N₄-R) at known concentration; spike them into samples.
• Output: True absolute amount (fmol/µg)
• Limit: Each target needs a custom SIS — expensive

二、Skyline & MSstats Targeted

# Skyline (https://skyline.ms) PRM/SRM 設計流程
Step 1 File → New → 選擇 PRM 或 SRM 模板
Step 2 Settings → Peptide Settings → 加入消化規則 (Trypsin), modifications
Step 3 Settings → Transition Settings:
        - Filter: charge 2+/3+, ion types b/y, 6 most intense
        - Library: import spectral library (e.g., from DDA 預掃)
Step 4 Edit → Insert → Peptide List
        - 貼上 candidate peptide 序列
        - Skyline 自動產生 transition list (Q1 → Q3 pairs)
Step 5 File → Export → Method
        - 輸出儀器方法檔 (Thermo .meth / Sciex .dam)
Step 6 跑樣後 → File → Import → Results
        - 自動 peak picking + integration
Step 7 Tools → External Tools → MSstats
        - 統計分析 (linear mixed model)
