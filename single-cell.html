單細胞蛋白質體學 (Single-cell Proteomics)

從 SCoPE-MS 到 timsTOF Ultra——單細胞蛋白質體學的演進、技術瓶頸與 2025 最新進展。

From SCoPE-MS to timsTOF Ultra — the evolution of single-cell proteomics, its bottlenecks, and 2025 state-of-the-art.

概念

為什麼單細胞蛋白質體學如此困難？

單個哺乳類細胞含 ~250 pg 蛋白質（vs bulk 樣品 1–100 µg），約為傳統樣品的 10⁶ 倍稀薄。挑戰包括：

  • 樣品損失：移液、容器吸附、消化失敗都會吃掉大半信號
  • 離子化效率：低濃度下 ESI 訊號顯著下降
  • 偵測極限：低豐度蛋白根本進不了 MS2 觸發閾值
  • 通量：單細胞數要夠才能做生物學分析

解決方案歷經數代演進，2025 已可從單細胞鑑定 5000+ 蛋白。

A single mammalian cell contains ~250 pg of protein (vs 1–100 µg in bulk) — 10⁶× more dilute. Challenges:

  • Sample loss: pipetting, tube adsorption, failed digestion can devour signal
  • Ionization: ESI efficiency drops at low concentrations
  • Detection limit: low-abundance proteins never trigger MS2
  • Throughput: need enough cells for meaningful biological analysis

Generations of solutions later, 5000+ proteins per single cell is achievable in 2025.

演進時間軸

一、技術演進時間軸

SCoPE-MS / SCoPE2 (Slavov lab)

用 TMT carrier channel（加入 200× 細胞當「carrier」）提升 fragment ion signal；單細胞當其中一個 channel。一次測 ~10 個單細胞，每個~1000 蛋白。

Uses a TMT carrier channel (200× cell-equivalent as carrier) to boost fragment ion signal; single cells become other channels. ~10 single cells per run, ~1000 proteins each.

nanoPOTS (Kelly lab)

在 nanowell chip（200 nL volume）做 SP3-like 消化，最小化樣品損失。可結合 LCM、Sort。

SP3-like digestion in nanowell chips (200 nL volume) to minimize loss. Integrates with LCM and FACS.

diaPASEF + timsTOF Pro 2

Bruker 結合 TIMS + DIA + PASEF，單細胞 ~2000-3000 蛋白。可實現 label-free single-cell。

Bruker's TIMS + DIA + PASEF combo enables ~2000–3000 proteins per single cell, with label-free workflows.

timsTOF Ultra / SCP

Bruker 為單細胞優化：更高靈敏度 ion source。報導單細胞最多 4000+ protein ID。

Bruker's single-cell-optimized platform with high-sensitivity ion source. Reports 4000+ protein IDs per single cell.

Orbitrap Astral + DIA

Thermo Astral 加入 single-cell 競賽。超高速 DIA + 高解析度 → 5000+ protein/cell，並可大規模 cohort (1000+ cells)。

Thermo's Astral joins the race. Ultra-fast DIA + high resolution → 5000+ proteins per cell at cohort scale (1000+ cells).

關鍵設計

二、實驗設計關鍵考量

🧊 樣品收集

FACS sort to 384-well 或 nanoPOTS chip。避免 PBS 連續沖洗（會吸附蛋白）；使用低吸附 tubing；冷凍儲存於 -80°C lyse buffer。

FACS sort into 384-well or nanoPOTS chip. Avoid serial PBS washes (protein adsorption); use low-bind tubing; cryostore in -80°C lyse buffer.

🧪 消化策略

用 DDM 或 sodium deoxycholate 取代 SDS；one-pot 消化避免轉移損失；Trypsin/LysC 1:25 比例，37°C 過夜。

Use DDM or sodium deoxycholate instead of SDS; one-pot digestion avoids transfer loss; trypsin/LysC at 1:25, 37°C overnight.

📡 MS 採集

label-free 用 diaPASEF 或 narrow-window DIA；標籤組合用 TMTpro 18-plex + carrier channel (SCoPE2)。LC 梯度 30–60 分鐘 (compromise of speed vs depth)。

Label-free: diaPASEF or narrow-window DIA. Labeled: TMTpro 18-plex + carrier (SCoPE2). LC gradient 30–60 min (speed-vs-depth balance).

📊 資料分析特殊性

缺失值極高（> 50%），需特殊 imputation 或 missing-aware 模型（如 SCP package, scp, R Bioconductor）。整合 single-cell UMAP / annotation 借鏡 scRNA-seq 套件。

Very high missingness (> 50%); requires specialized imputation or missing-aware models (scp, R Bioconductor). UMAP / annotation borrow scRNA-seq tooling.

實作

三、scp 套件範例

library(scp)
library(QFeatures)

# Load PSM-level data (e.g., from MaxQuant)
scp <- readSCP(quantTable, metaTable,
                channelCol = "Channel",
                batchCol   = "Run")

# Filter: PSM-level QC
scp <- filterFeatures(scp, ~ Reverse != "+" & PEP < 0.02)

# Compute SCR (sample-to-carrier ratio) per PSM
scp <- computeSCR(scp, i = 1:10, colvar = "SampleType",
                  samplePattern = "Single",
                  carrierPattern = "Carrier")

# Aggregate to peptides → proteins
scp <- aggregateFeaturesOverAssays(scp, i = 1:10,
          fcol = "Sequence", name = paste0("peptides_", 1:10))
scp <- joinAssays(scp, i = paste0("peptides_", 1:10), name = "peptides")
scp <- aggregateFeatures(scp, i = "peptides", name = "proteins",
                        fcol = "Leading.razor.protein", fun = colMedians)

# UMAP for single-cell visualization
library(uwot)
mat <- assay(scp[["proteins"]])
mat[is.na(mat)] <- 0
um <- umap(t(mat), n_neighbors = 15)
plot(um[,1], um[,2], col = factor(meta$cellType))
import scanpy as sc   # reuse scRNA-seq tooling
import anndata as ad
import pandas as pd

# Build AnnData from protein matrix (cells × proteins)
prot = pd.read_csv("proteins.csv", index_col=0).T
meta = pd.read_csv("cell_meta.csv", index_col=0)
adata = ad.AnnData(X=prot.values, obs=meta, var=pd.DataFrame(index=prot.columns))

# Filter cells & proteins by missingness
sc.pp.filter_cells(adata, min_genes=800)
sc.pp.filter_genes(adata, min_cells=int(0.3 * adata.n_obs))

# Log-transform & scale
sc.pp.log1p(adata)
sc.pp.scale(adata, max_value=10)

# PCA → UMAP → Leiden
sc.tl.pca(adata, n_comps=30)
sc.pp.neighbors(adata, n_neighbors=15)
sc.tl.umap(adata)
sc.tl.leiden(adata, resolution=0.6)
sc.pl.umap(adata, color="leiden")

A. 提供同位素校準A. Isotope calibration

C. 替代 DDAC. Replaces DDA
D. 用於 QCD. For QC purposes

2. nanoPOTS's core advantage?

A. 在 200 nL nanowell 進行消化，最小化轉移損失與容器吸附A. Digestion in 200 nL nanowells minimizes transfer loss and tube adsorption
B. 不需要 LCB. No LC needed
C. 不需要 MSC. No MS needed
D. 完全自動化D. Fully automated

3. Why is single-cell proteomics missingness so high?

A. 細胞太多A. Too many cells
B. 質譜儀故障率高B. MS instruments fail often
C. 蛋白量極低，低於 LOD 的蛋白本來就比例高（典型 MNAR），加上 DDA stochastic 缺失C. Protein amounts are very low, so many fall below LOD (typical MNAR) — compounded by DDA stochastic missing
D. 無法用 TrypsinD. Can't use trypsin
